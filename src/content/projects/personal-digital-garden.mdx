## Masalah

Blog tradisional bersifat kronologis — tulisan lama tenggelam, tidak ada hubungan antar catatan, dan tidak mendorong penulis untuk memperbaiki tulisan lama. Saya butuh tempat yang lebih organik untuk menumbuhkan pengetahuan.



![](/images/inline/Cae.PNG)

## Solusi

Membangun **digital garden** — sebuah website personal di mana catatan bisa saling terhubung melalui \[\[Membangun Digital Garden|wiki links]], tumbuh dari seedling menjadi evergreen, dan diorganisasi berdasarkan topik bukan tanggal.

## Teknologi

* **\[\[Belajar Astro Framework|Astro]]** — static site generator dengan island architecture
* **TypeScript** — type safety end-to-end
* **\[\[Eksperimen Tailwind CSS v4|Tailwind CSS v4]]** — utility-first styling
* **MDX** — markdown dengan komponen interaktif
* **Pagefind** — client-side full-text search

## Fitur yang Sudah Diimplementasi

* ✅ Content Collections dengan schema validation (Zod)
* ✅ Wiki links (`[[Judul]]`) dengan resolusi otomatis
* ✅ Backlinks di setiap halaman konten
* ✅ Topic/tag system dengan halaman per topic
* ✅ Growth stages (seedling → budding → evergreen)
* ✅ Client-side filtering di Garden index
* ✅ Dark mode dengan tema Claude (charcoal + orange accent)
* ✅ Full-text search (Pagefind)
* ✅ RSS feed & sitemap
* ✅ Responsive design
* ✅ SEO & Open Graph metadata

## Tantangan

1. **Wiki link resolution** — butuh pre-build step (`generate-links.ts`) yang membaca semua MDX, extract frontmatter + aliases, dan generate link map JSON sebelum Astro build.
2. **Pagefind + Vite** — dynamic import harus di-wrap dengan `@vite-ignore` karena Pagefind files baru ada setelah build.
3. **Dark mode di static site** — harus inject inline script di `<head>` sebelum render untuk prevent flash of wrong theme.

## Pelajaran

1. **Astro Content Collections** sangat powerful untuk validasi schema — error muncul saat build, bukan di runtime.
2. **Static output** memungkinkan hosting gratis dan performa sangat cepat.
3. **Digital garden** sebagai format lebih cocok untuk learner — catatan boleh belum sempurna.

## Arsitektur

```text
src/
├── content/          # MDX content (notes, essays, projects, experiments)
├── components/       # Astro components (layout, content, garden)
├── layouts/          # Page layouts (Base, Content, Project, Page)
├── lib/              # Utilities & helpers (content, links)
├── plugins/          # Remark plugins (wiki-link)
├── data/             # Generated data (links.json)
└── styles/           # Global CSS & design tokens
```
