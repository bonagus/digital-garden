## Kenapa Astro?

Astro menarik karena konsep **island architecture** — hanya mengirim JavaScript ke browser saat benar-benar dibutuhkan.

Ini cocok untuk [[Membangun Digital Garden]] karena konten statis tidak butuh client-side JS yang berat.

## Fitur yang Menarik

1. **Content Collections** — validasi frontmatter pakai Zod
2. **MDX support** — bisa embed komponen di markdown
3. **Static output** — bisa deploy ke mana saja
4. **View Transitions** — animasi halus tanpa SPA

## Hubungan dengan Project

Saya menggunakan Astro untuk membangun [[Personal Digital Garden]] ini. Stack lengkapnya bisa dilihat di catatan tentang [[Eksperimen Tailwind CSS v4]].
